Техническое задание (Backend-only) | Проект: AI Mafia
1. Стек технологий
Фреймворк: Python 3.11+ / FastAPI

Транспорт: REST API (для создания комнат) + WebSockets (для реалтайм-игры и чата)

Хранилище состояний (State Management): Redis (для хранения состояния комнат, истории чатов, таймеров) или in-memory хранилище (на этапе MVP)

AI Провайдер: OpenRouter API (интеграция через httpx для асинхронности)

Управление конфигурацией: pydantic-settings (работа с .env)

2. Подключение OpenRouter через .env (Требование)
Взаимодействие с нейросетями идет исключительно через сервис OpenRouter для доступа к топовым моделям (Claude 3, GPT-4o, Llama 3) с единым API.

В корне проекта лежит файл .env:

Фрагмент кода
OPENROUTER_API_KEY=sk-or-v1-xxxxxxxxxxxxxxxxx
DEFAULT_AI_MODEL=meta-llama/llama-3-70b-instruct
ENVIRONMENT=development
Бэкенд использует BaseSettings из Pydantic для валидации переменных окружения при старте сервера.

3. Новые "Фишки" (Дополнения к логике)
Тайпинг-имитация (Typing Indicator): Когда AI-агент "думает" и генерирует ответ, бэкенд отправляет по WebSocket событие agent_is_typing. Сами сообщения от AI можно отдавать стримингом (по словам), чтобы это выглядело как реальный набор текста человеком.

Система "Характеров" (Prompt-профили): Чтобы агенты не были одинаковыми ботами, при инициализации комнаты бэкенд рандомно присваивает каждому AI скрытый промпт-характер (например: "Параноик — всех подозревает", "Молчун — пишет коротко", "Агрессор — сразу давит на случайного игрока").

Координатор диалога (Очередь сообщений): Чтобы 4 AI-агента не отправляли сообщения в одну секунду, бэкенд реализует менеджер диалога. Он вводит случайные задержки (delay) перед ответом агента в зависимости от длины предыдущего сообщения в чате.

Spectator & Dead Chat: Игроки, которых убили, отключаются от основного чата и переводятся в "Чат призраков" (Ghost Chat), где могут свободно обсуждать игру, не влияя на живых. Туда же подключаются зрители.

Реконнект по JWT/Session ID: Если у игрока-человека моргнул интернет, он должен иметь возможность переподключиться к своему WebSocket соединению с помощью токена сессии, не ломая state машины.

4. Архитектура и Эндпоинты
REST API (Управление лобби)
POST /api/rooms — Создать комнату. Возвращает room_id, host_token. Принимает настройки (роли, количество AI/людей, модель поведения).

GET /api/rooms/{room_id} — Получить публичный статус комнаты (сколько занято слотов, идет ли игра).

POST /api/rooms/{room_id}/join — Регистрация игрока (ввод никнейма). Возвращает player_id и session_token для подключения к WebSocket.

WebSocket (Игровой процесс)
ws://{domain}/api/ws/rooms/{room_id}?token={session_token}

События от Клиента (Client -> Server):

chat_message: Отправка сообщения в чат.

vote_action: Голосование за игрока (днем) или выбор жертвы/проверки (ночью).

start_game: Команда от хоста на старт (если собрано 5+ игроков).

События от Сервера (Server -> Client):

game_state_update: Обновление фазы (Лобби -> Распределение ролей -> Ночь -> День).

chat_event: Новое сообщение в чате (от человека или AI).

timer_update: Тик таймера текущей фазы.

personal_info: Приватная информация (вы — мафия, ваши союзники — игроки 2 и 4).

reveal_endgame: Итоги игры со статистикой кто был AI, а кто человеком.

5. Жизненный цикл игры (State Machine на Backend)
Lobby State: Ожидание подключения по WebSocket. Если хост жмет "Старт" при недостатке людей (например, 3 человека из 5), бэкенд автоматически создает инстансы AI-агентов, генерирует им ники и подключает в комнату.

Role Assignment: Бэкенд тасует роли (Мирные, Мафия, Доктор, Комиссар) и рассылает WebSocket эвенты personal_info. AI-агенты получают свои роли в системный промпт.

Night Phase: * Люди-активы видят UI для действий.

Мафия (люди и AI) общаются в отдельном ночном чате (для AI это отдельная ветка контекста).

Бэкенд запрашивает у AI-агентов (у кого активные роли) их действия через структурированный JSON-ответ от OpenRouter.

Day Phase: * Бэкенд транслирует всем результаты ночи.

Открывается общий чат. Бэкенд собирает историю сообщений.

Когда приходит очередь AI генерировать мысль (или если AI "триггерится" на сообщение человека), бэкенд формирует запрос: [Системный промпт с правилами и ролью] + [История чата] -> Генерация ответа.

Voting Phase: Подсчет голосов. Для AI-агентов бэкенд делает скрытый запрос к LLM: "Проанализируй чат, кто по твоему мнению мафия? Верни JSON с ID игрока".

Turing Test Phase (Post-game): Игра закончена. Бэкенд отправляет форму голосования людям: "Отметьте тех, кого считаете AI". Подсчет метрики "человечности" ботов. Раскрытие правды.

6. Взаимодействие с OpenRouter (Паттерн для AI)
Чтобы игра не тормозила, используется фоновый asyncio.Task для каждого AI-агента.

Пример логики запроса (псевдо-код):
